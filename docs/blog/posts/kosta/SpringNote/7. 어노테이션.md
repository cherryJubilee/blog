# 자바 개발 어노테이션

> 자바 개발 어노테이션

<!-- more -->

### 1. 어노테이션의 역할

1. 어노테이션은 자바에서 특별한 의미나 역할, 기능을 수행할 수 있도록 해주는 역할을 함.

### 2. 기본 Web 개발 어노테이션

-   `@Component`

    -   해당 클래스가 스프링의 컴포넌트로 등록된다는 것을 나타냄.
    -   일반적으로 개발자가 만든 클래스를 객체(Bean)로 등록할 때 사용함.
    -   스프링이 관리하는 모든 컴포넌트에 대한 제너릭 스테레오타입임.

-   `@Controller`
    -   해당 클래스가 Controller 역할을 한다는 것을 나타냄.
    -   일반적으로 요청을 받아서 처리하고 응답하는 역할을 함.
    -   @Component의 특수한 형태로, 프리젠테이션 계층에서 사용하면 적합함.

### 3. 클라이언트 요청/응답 파라미터 관련 어노테이션

-   `@RequestMapping(value = "패스")`
    -   클라이언트에서 들어온 요청이 `value`의 패스와 동일하면 해당 클래스나 메서드를 실행함.
    -   클래스 단위에서 사용 가능하며, 매핑될 URL의 공통 부분을 클래스에 적용함.
    -   메서드 단위에서도 사용 가능함.

### 4. API 개발 관련 어노테이션

-   `@GetMapping`

    -   GET 요청을 처리하기 위한 메서드를 지정함.

-   `@PostMapping`
    -   POST 요청을 처리하기 위한 메서드를 지정함.
